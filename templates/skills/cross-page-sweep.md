# cross-page-sweep

The horizontal corpus-dimension scope-widen · sibling to /core-loop's vertical layer-stack pivot.

## When to invoke

```
trigger conditions
  stuck on a single page / single instance
  pattern hard-to-see in the stuck instance because complexity confounds
  single-instance hypothesis · need evidence not just hypothesis
  /core-loop step 3b "scope-widen once" needs operationalization
  before declaring an authoring-gap r34 carrier · check if same gap exists
    elsewhere (might be a populator bug not an authoring gap)
```

## The loop

```
1.  NAME the pattern                  at the stuck instance · syntactic
                                      signature you can probe for

2.  SWEEP across all instances        per-instance mechanical probe ·
                                      grep · query · count

3.  RANK by visibility                page complexity · confound density ·
                                      instance count · pick K simpler instances

4.  TRIANGULATE                       pattern that holds across N is structural ·
                                      holds-for-one is incidental

5.  AUTHOR the fix                    where simpler instances prove it cleanly

6.  PROPAGATE back to stuck           re-validate at the original instance
```

## Why this works

- complex pages have CONFOUNDS that obscure the pattern
- simpler pages with the same pattern reveal the SIGNATURE without confounds
- multiple instances triangulate a hypothesis into evidence
- authoring at the simplest passing instance is cheaper than authoring at the stuck instance

## Key insights

- puzzle-piecing via cross-instance corroboration
- CORPUS-dimension scope-widen (different from /core-loop's vertical layer-stack · this is horizontal at the same level)

- simpler-instance-first heuristic · less complexity = pattern more visible · confounds fewer = signal cleaner
- catches the case where "structural pivot" signal is actually a single-instance bug · NOT authoring gap
