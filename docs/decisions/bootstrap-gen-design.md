# bootstrap: Consumer template generation + harness

## Problem

Consumers of the roadmap library need to:
1. **Scaffold** a new project roadmap (minimal INIT → TERM)
2. **Boot** an executor agent (auto-orient, position, confirm)
3. **Persist** roadmap metadata for replay/debugging

Current state: protocols exist (define, check, verify, orient, merge), but consumers have no template or harness to start from. Each consumer builds these independently, risking incomplete contracts or bad initialization order.

## Solution: bootstrap command + template

Generate three artifacts:

| Artifact | Purpose | Who reads |
|----------|---------|-----------|
| `roadmap.ts` | Minimal DAG: INIT (what exists) + TERM (goal) | Consumer project; consumed by orient/check |
| `boot.ts` | Entry point: validate, orient, report readiness | Agent at session start |
| `.roadmap/` | Metadata directory: checkpoints, git history, receipts | bootstrap logic + debugging |

## Roadmap template schema

Minimal consumer roadmap.ts generated by bootstrap command:

```typescript
import { define, graph } from 'roadmap/protocol';

export default define(graph({
  id: '{project-name}',
  desc: '{project-desc}',
  init: 'init',
  term: 'deployed',
  nodes: {
    init: {
      id: 'init',
      desc: 'What exists now',
      produces: [
        'package.json',
        'src/index.ts',
      ],
      consumes: [],
      deps: [],
    },
    deployed: {
      id: 'deployed',
      desc: 'What should exist',
      produces: [],
      consumes: [
        'package.json',
        'src/index.ts',
        'dist/index.js',
        'dist/index.d.ts',
      ],
      deps: ['build'],
    },
    build: {
      id: 'build',
      desc: 'Compile + generate types',
      produces: [
        'dist/index.js',
        'dist/index.d.ts',
      ],
      consumes: [
        'package.json',
        'src/index.ts',
      ],
      deps: ['init'],
    },
  },
}));
```

### Key constraints
- **init**: must list actual artifacts (checked by orient via filesystem)
- **term**: consumes = union of all produces up-chain
- **deps**: acyclic (define() catches cycles)
- **validation hooks**: required property for each node (checked at node completion)
- **idempotent**: optional boolean — if true, node can be re-run safely (deterministic produces)

#### Idempotency semantics

Idempotent nodes enable recovery: lost artifacts can be recreated by re-running the node.

| Type | Idempotent | Recovery |
|------|-----------|----------|
| Write source file | ✓ | Deterministic from spec → re-write, validation proves correctness |
| Run tests | ✓ | Same code → same results; re-run proves no regression |
| Generate docs | ✓ | From DAG structure; always reproducible |
| Approve PR (manual) | ✗ | Requires human decision; tracked in audit trail |
| Deploy to prod | ✗ | State changes; separate audit trail, checkpoint/restore |

Strategy:
- All spec/code/test nodes marked `idempotent: true`
- Manual/external nodes marked `idempotent: false` (or undefined)
- Validation failure on idempotent node → auto-recovery: re-run node, re-validate, prove recovery
- Validation failure on non-idempotent → requires manual intervention or rollback

## Boot harness (boot.ts)

Entry point for autonomous agents:

```typescript
import { readHeadDAG } from './.roadmap/query.ts';
import { orient, check, verify } from 'roadmap/protocol';
import { existsSync } from 'node:fs';
import { join } from 'node:path';

export async function boot() {
  const dag = await readHeadDAG(process.cwd());

  // Validate DAG integrity
  const checkResult = check(dag);
  if (!checkResult.done) {
    throw new Error(`DAG not fully connected: ${JSON.stringify(checkResult.orphans)}`);
  }

  const verifyErrors = verify(dag);
  if (verifyErrors.length) {
    throw new Error(`Contract violations: ${JSON.stringify(verifyErrors)}`);
  }

  // Find current position
  const fsCheck = (a: string) => existsSync(join(process.cwd(), a));
  const position = orient(dag, fsCheck);

  // Report readiness
  return {
    ready: true,
    position: position.position,
    produces: position.produces,
    consumes: position.consumes,
    remaining: position.remaining.length,
  };
}
```

## .roadmap/ metadata structure

```
.roadmap/
├── head.json                 # Current DAG (git-tracked checkpoint)
├── query.ts                  # readHeadDAG + getReconciliationManifest
├── checkpoints/              # History of roadmap versions
│   ├── v0.1.0.json
│   └── v0.2.0.json
└── receipts/                 # Agent execution logs
    ├── session-{id}.json
    └── manifest-{node-id}.json
```

### head.json schema
```json
{
  "id": "project-name",
  "desc": "project description",
  "init": "init",
  "term": "deployed",
  "timestamp": "2026-02-25T10:00:00Z",
  "nodes": { /* full DAG */ }
}
```

### session-{id}.json schema (written by agent after completion)
```json
{
  "sessionId": "boot-{timestamp}",
  "position": "build",
  "status": "complete",
  "artifacts": [
    { "path": "dist/index.js", "hash": "sha256:..." },
    { "path": "dist/index.d.ts", "hash": "sha256:..." }
  ],
  "duration": 1234,
  "nextNode": "deployed"
}
```

## Generation strategy

Bootstrap command generates template by inspecting consumer package:

```bash
npx roadmap bootstrap
  --project-name my-project
  --project-desc "What this builds"
  --init-artifacts src/index.ts,package.json
  --term-artifacts dist/index.js,dist/index.d.ts
```

CLI logic:
1. Validate init-artifacts exist (abort if missing)
2. Infer build steps (tsc → dist, rollup, etc.) based on package.json
3. Generate roadmap.ts with suggested nodes
4. Create .roadmap/ directory + head.json
5. Output boot.ts + query.ts
6. Print validation result (check + verify)

## Integration points

### Agent lifecycle
```typescript
// Session start
const receipt = await boot();
if (!receipt.ready) {
  console.error('Roadmap not ready:', receipt);
  process.exit(1);
}

// Work loop
const produces = receipt.produces;
// create files...

// Session end
await recordReceipt(receipt.sessionId, {
  artifacts: produces.map(p => ({ path: p, hash: gitHash(p) })),
});
```

### Multi-repo coordination
merge() + bootstrap work together:

```typescript
// Project A has its own roadmap
const projA = await readHeadDAG('/path/to/proj-a');

// Project B is a dependency, also has roadmap
const projB = await readHeadDAG('/path/to/proj-b');

// Merge at shared output artifact
const combined = merge(projA, projB, [
  { g1Node: 'deployed', g2Node: 'init', artifact: 'dist/proj-a.js' }
]);

// Parent orchestrator uses merged DAG
const pos = orient(combined, exists);
```

## Testing strategy

Adversarial tests (bootstrap-gen.test.ts):

| Scenario | Test | Expectation |
|----------|------|------------|
| Init validation | Init node lists non-existent files | boot() fails early |
| DAG cycles | Create cycle in roadmap.ts | define() throws |
| Missing nodes | init→term not connected | check() returns orphans |
| Empty produces | Node with produces: [] | orient() handles correctly (not stalled) |
| Parallel bootstrap | Two agents boot same repo | Both get consistent position |
| Checkpoint restore | Save state, boot with old roadmap | orient() reflects old position |
